    Overview

    Initial TMS Installation

    1. Use these instructions only if you are a first-time installer of Revenova TMS. See the Upgrading from a Previous Version section to perform an upgrade of a previous release.

    2. From Setup → Lightning Email Templates toggle Folders and Enhanced Sharing to On.

      1. Create a new Permission Set.

        1. From Setup → Permission Sets.

        2. Click the New button at the top of the Permission Sets list.

        3. Label the new Permission Set, Manage Public Lightning Email Templates and Drag-and-Drop Content Builder.

        4. Click Save.

        5. Click the link for System Permissions and click the Edit button. Search for these two permissions and add them to the Permission Set.

          1. Manage Public Lightning Email Templates.

          2. Access drag-and-drop content builder.

        6. On the Permission Changes Confirmation modal, click Save.

      2. Assign the Manage Public Lightning Email Templates and Drag-and-Drop Content Builder Permission Set to the logged in user completing the installation.

    3. Obtain the URL and install the package, using the Install for all Users setting.

    4. Wait a few minutes for the application to complete the installation process. You will receive an email upon completion of the installation.

    5. From Setup → Installed Packages click on the Manage Licenses link for the TMS package and assign licenses to users.

    6. From Setup → State and Country/Territory Picklists turn on Country/State Picklists. Start with the United States as the only country unless you have more information.

    7. From Setup → Data → Data Integration Rules activate Geocodes for Account Shipping Address.

    8. From Setup → Enhanced Email. Confirm that Enhanced Email is Enabled.

    9. From Setup → Workflow Rules. Make sure the TMS Auto-Reject Carrier Transportation Profile workflow rule is active or an equivalent method to invalidate unacceptable carriers.

    10. From Setup → Duplicate Rules. Turn off Matching Rules and Duplicate Rules for Accounts and Contacts as they can prevent auto-creation of accounts in the application.

    11. From Setup → Session Settings. Check the Use Lightning Web Security for Lightning web components and Aura components setting to ensure all components meet Salesforce security standards. Click Save.

    12. From Object Manager, navigate to the following objects. For each, click the Edit button. Under Optional Features, check the box for Track Field History. Click Save.

      1. Carrier Quote

      2. Carrier Quote Accessorial

      3. Claim

      4. Container

      5. Customer Quote

      6. Customer Quote Accessorial

      7. Customer Quote Tax

      8. Driver

      9. EPayment

      10. Fleet Asset

      11. Line Item

      12. Load

      13. Load Accessorial

      14. Load Reference

      15. Sales Tax

      16. Stop

      17. Transportation Profile

    13. Enable Feed Tracking for the following objects. This enables the Chatter feature across Fleet Management. (Only enable for Fleet Management)

      1. From Setup → Feed Tracking, select to the following objects.

      2. Check Enable Feed Tracking.

      3. Click Save.

        1. Driver

        2. Fleet Asset

        3. Load

        4. Trip Plan

    14. From TMS Admin → Configuration click on the Reset Reference Data button ONCE. Wait for this process to complete.

    15. From TMS Admin → Configuration fill out all the fields in the top section with valid values.

    16. Create a global (broker) transportation profile as the global default and set it from TMS Admin → Configuration.

    17. From TMS Admin → Equipment Types set the four load board equipment types by visiting and filling out all information on each Equipment Type record.

    18. From Setup → Chatter Settings enable Chatter.

    19. From Setup → Trusted URLs add a new trusted URL for ALK Maps:

    20. Make sure these custom Account fields are displayed for TMS users, properly secured.

      1. TMS Type

      2. Transportation Profile

      3. DBA Name

      4. Customer Sales Rep

      5. Default Payment Method

      6. Posting City

      7. Posting State/Province

      8. Time zone

    21. Make sure the Transportation Role field is displayed for TMS users on the Contact page layout.

    22. Provide appropriate CRUD access to each user profile for each TMS object.

    23. Tailor page layouts and lightning record pages as appropriate for each profile.

    24. Salesforce requires that user email addresses are verified to enhance accountability and security when sending emails. When an org is updated to the Salesforce Spring ‘25 release, a reminder email is no longer sent. If encountering issues with users ability to send email messages, confirm the email address is verified. The Getting Started with Revenova article contains steps to confirm an email address is verified. See additional Salesforce information here.

    25. Please refer to Getting Started with Revenova and the Revenova Knowledge Base for detailed descriptions, configuration settings, and customizations available in the application.

    Upgrading from a Previous Version

    You must perform each set of upgrade tasks (EXCEPT for the package installation step which can skip from any release to the latest) if you are upgrading from a release prior to the previous release.

    Follow the upgrade steps from the oldest release to the newest release, in order, for any upgrades that span multiple releases.

    Release Spring ‘25 TMS Upgrade Instructions (ETC: 45 minutes)

    • Coordinate with your EDI VAN to confirm they have implemented the latest version of the Revenova EDI Specification. The VANs must be on the latest version even if not using the Rail functionality.

    • Thoroughly test ALL EDI Transactions in the sandbox before upgrading to production. EDI enhancements included in the Spring ‘25 release impacts all EDI transactions.

    Accounting Package Compatibility

    Prior to TMS package installation, confirm there is a compatible accounting package version for the options below.

    • Sage API (Sage 300)

    • Sage Flat File

    • Dynamics GP

    Contact Professional Services with any questions.

    Accounting Seed customers must have BOTH the TMS and Accounting Seed Bridge packages before upgrading. Complete the TMS Upgrade installation before installing the Accounting Seed Bridge package. See the Accounting Seed Bridge installation instructions for more information.

    Items in blue are optional and may be skipped if the feature is not used. Structural changes in red are required.

    1. You must install and test in a sandbox environment before upgrading to the production environment.

    2. You must have installed and completely configured the Winter ‘25 Release before installing this release.

    3. Salesforce requires that user email addresses are verified to enhance accountability and security when sending emails. When an org is updated to the Salesforce Spring ‘25 release, a reminder email is no longer sent. If encountering issues with users ability to send email messages, confirm the email address is verified. The Getting Started with Revenova article contains steps to confirm an email address is verified. See additional Salesforce information here.

    4. SFDC/TMS Pre-Configuration

      1. For Accounting Seed customers, do not install this TMS release until you have the Accounting Seed Bridge package URL also. They must be upgraded at the same time.

      2. From Setup → Duplicate Rules. Turn off Matching Rules and Duplicate Rules for Accounts and Contacts as they can prevent auto-creation of accounts in the application.

      3. From Setup → Data → Data Integration Rules activate Geocodes for Account Shipping Address.

      4. From Setup → State and Country/Territory Picklists turn on Country/State Picklists. Start with the United States as the only country unless you have more information. These picklists should be enabled unless there is a backward-compatibility issue with other packages or customizations.

      5. From Setup → Session Settings. Check the Use Lightning Web Security for Lightning web components and Aura components setting to ensure all components meet Salesforce security standards. Click Save.

        Thoroughly test any custom Lightning Web Components when enabling this setting. All TMS components are tested to support this functionality.

      6. From the Object Manager, make sure no values were added to the Load object’s Sales Status or Load Status picklist fields or the Stop object’s Stop Status picklist field.

        For trouble deleting items in the Lightning UI, change to Classic UI to perform the deletions.

    5. The Carrier Quote Tender Declined Reason field is removed in the Spring ‘25 package installation. Any custom picklist values added to this field must be added to the new Carrier Quote Tender Declined Reasons field.

      1. Navigate to Setup → Object Manager.

      2. Choose the Carrier Quote object.

      3. Select the Fields & Relationships tab.

      4. Choose the Tender Declined Reason field.

      5. The original picklist values include: Equipment Breakdown, Equipment No Longer Available, and Other. Note any other custom values and save for post-package steps.

      6. See later steps for more information.

    6. Obtain the Spring ‘25 installation URL and password from your Revenova representative.

    7. Install the upgrade.

      1. Select Install for All Users.

      2. Click Upgrade.

      3. Check the Yes, grant access to these third-party web sites box.

      4. Click Continue.

      5. In Setup, find Deployment Status and click the link to view the progress of the installation.

      6. Wait for the Installation succeeded email.

        Several new objects and fields are included in the TMS Spring ‘25 release. Set CRUD access for the objects and/or fields accordingly.

        Warning

        Upgrade installations do not add any objects or fields to custom permission sets. Add new fields as necessary to any custom permission sets. New fields are added automatically to Revenova packaged permission sets and do not require additional setup.

        Custom permission sets are commonly used for the following.

        Carrier Community

        Carrier Email Loop Customer Community

        Customer Email Loop

        Fleet Email Loop

    8. From TMS Admin → Configuration click on the Reset Reference Data button ONCE and wait for the email indicating completion. This process completes the following tasks.

      1. Sets the User Distance Units in Trip Planner field to Miles and Kilometers for all users.

      2. Updates the following fields for use by the web service. (Object → Field)

        1. Lane → Destination Value (Indexed).

        2. Lane → Origin Value (Indexed).

        3. Zone Entry → Value (Indexed).

      3. Sets all Transportation Profile Automatic Carrier Tracking fields to Global.

      4. Inserts the following picklist values. (Object → Field → Picklist Value)

        1. EDI Transaction → Transaction Type → 322, 404, 410, 824, 998.

        2. Fuel Card → Fuel Card Provider → Comdata.

        3. Unmatched EDI Transaction → Type → EDI 410 Invoice.

        Any errors encountered while running the Release Update Job are sent to the email account for the User running the job. Errors can also be found in the Apex Jobs view.

    9. From Setup → Picklist Value Sets, find the Movement Types picklist value set.

      1. Select the Movement Types picklist value set and add the following picklist values.

        1. Motor/Rail - Ramp to Ramp

        2. Patron/Customer - Door to Destination Ramp

        3. Patron/Customer - Door to Door

        4. Patron/Customer - Origin Ramp to Door

        5. Patron/Customer - Ramp to Ramp

        6. Rail - Door to Destination Ramp

        7. Rail - Door to Door

        8. Rail - Origin Ramp to Door

        9. Rail - Ramp to Ramp

        10. Steamship/Stack Operator - Door to Destination Ramp

        11. Steamship/Stack Operator - Door to Door

        12. Steamship/Stack Operator - International - Door to Destination Ramp

        13. Steamship/Stack Operator - International - Door to Door

        14. Steamship/Stack Operator - International - Origin Ramp to Door

        15. Steamship/Stack Operator - International - Ramp to Ramp

        16. Steamship/Stack Operator - Origin Ramp to Door

        17. Steamship/Stack Operator - Ramp to Ramp

    10. Remove the following obsolete field that was deleted from the TMS. (Complete this step in Salesforce Classic mode).

      1. Navigate to Setup and search for and select Objects.

      2. Choose the Carrier Quote object.

      3. Click the Del link for the Tender Declined Reason field.

      4. Return to the Salesforce Lightning Experience.

    11. The Lane Upsert and Load Posting trigger architecture is enhanced. Thoroughly test that any custom code based on these triggers is working properly when updating to the Spring ‘25 release.

    12. Continue the process from Step 5 in the installation. Do not delete any picklist values. If no custom picklist values were added to the Tender Declined Reason field, skip this step.

      1. Investigate new picklist values.

        1. Navigate to Setup → Object Manager.

        2. Choose the Carrier Quote object.

        3. Select the Fields & Relationships tab.

        4. Choose the Tender Declined Reasons field.

        5. Review the picklist options added to the Tender Declined Reasons field during package installation. New picklist values may account for the old custom values. If the new picklist values do not account for the obsolete Tender Declined Reason field custom picklist values, re-add custom values as necessary.

      2. Reports and Dashboards that use the old field must be updated to include the new field for reporting. See Salesforce Report documentation for more information.

    13. Complete the following steps for automatic Load Tracking enhancements.

      1. Add the following fields to the Load Lightning Record Page by completing the following steps.

        1. Navigate to a Load page and choose Setup → Edit Page.

        2. Drag the following fields into Tracking Information section (on the default Revenova Lightning Record Page). For custom Load LRPs, add these fields to a section with other tracking fields.

          1. Automatic Carrier Tracking

          2. Carrier Tracking Status

          3. Carrier Accepted Tracking

        3. Click Save.

      2. Add the following fields to the Transportation Profile Lightning Record Page by completing the following steps.

        1. Navigate to a customer Transportation Profile page and choose Setup → Edit Page.

        2. Drag the following fields into the Carrier Tracking Administration section (on the default Revenova Lightning Record Page). For custom Transportation Profile LRPs, add these fields to a section with other Carrier tracking fields.

          1. Automatic Carrier Tracking

          2. Automatic Tracking Start Trigger

          3. Automatic Carrier Tracking Enabled Modes

          4. Tracking Provider

          5. Tracking Duration

          6. Tracking Update Interval

        3. Remove the following field from the page layout.

          1. Find the Carrier Tracking Provider field and remove it from the page.

        4. Click Save.

    14. The Carrier & Customer Quoting/Rate Entry auditing logs feature requires adding 2 new LWCs to the Load Lightning Record Page. The upgrade installation adds these two components to the Revenova default pages.

      1. Navigate to a Load record.

      2. From the Setup icon, select Edit Page.

      3. Select the System tab below the Load Banner LWC.

      4. Drag the apiQuoteLog component onto the page below the Load Emails component, or where desired.

      5. Drag the loadMatchTest component onto the page below the API Quote Log component, or where desired.

      6. Click Save.

      7. Add the API Quote Logging field to the desired Load page layout to make the field editable in list views.

        1. Navigate to Setup → Object Manager and select the Load object.

        2. Select the Page Layouts tab. Choose the appropriate Load page layout.

        3. Add the API Quote Logging field to the page layout.

        4. Click Save.

    15. The TextLocate LWC is added automatically to the Revenova Load Lightning Record Page. Customers using a custom Load page and want to add use the TextLocate integration must add the new LWC to their Load Lightning Record Page.

      1. Navigate to a Load and select Setup → Edit Page.

      2. Select the Tabs component.

        1. On the right panel, click the Add Tab button.

        2. Enter “TextLocate” into the Custom Label field.

        3. Click Done.

      3. Choose the new TextLocate tab.

      4. Drag the TextLocate LWC onto tab.

      5. Save the page.

    16. Only customers using LaneIQ Network may use the Market Monitor LWC. Complete the following steps to add the LWC to the Home and Load LRPs as necessary.

      1. Orgs using the Revenova default Home and Load Lightning Record Pages may skip these steps. The upgrade installation adds the Market Monitor LWC to the Revenova default pages.

      2. Follow the steps below to add a new tab with the Market Monitor component.

        1. Navigate to the org. Home page and select Setup → Edit Page.

          1. On the right side of the page, find the Batch Process Monitor Default Tab.

          2. Below the tab names, click the Add Tab button.

          3. Select the Custom option for the Tab Label. Enter Market Monitor and click Done.

          4. Choose the new tab on the page. Drag the Market Monitor custom component into the “Add Component(s) Here" section.

          5. Save the page.

          6. If the customer requests another location on the Home page, add the component to that location instead.

        2. Depending on customer customization of the Home page, they may not have tabs with components like the Batch Process Monitor, Performance IQ, and/or Lane IQ. In that case, add the Market Monitor component to the customer’s desired location.

      3. Navigate to the Load page and select Setup → Edit Page.

        1. Find the Snapshot tab on the left side of the page.

        2. Drag the Market Monitor custom component onto the page below the RateIQ component, or another desired location.

          1. The following component visibility settings are added to the default component. Add the following if desired.

            1. User > Display Optional Features Equal true

            2. Record > Load Status Equal Unassigned

            3. Record > Load Status Equal Quotes Requested

            4. Record > Load Status Equal Quotes Received

            5. Record > Load Status Equal Tendered

        3. Save the page.

    17. Configure the Carrier Guest LWC with the following options for customers using the RFQ Load Board available in the Carrier Email Loop.

      1. Add the new field to the Transportation Profile Lightning Record Page.

        1. From a Carrier Transportation Profile select Setup → Edit Page.

        2. Add the Enable RFQ Load Board field to the page with other Load Posting fields.

        3. Click Save.

      2. Check the Transportation Profile Enable RFQ Load Board field for each Carrier that may access the RFQ Load Board.

      3. From Setup, search for and select All Sites.

        1. Click the Builder link for the Email_Loop.

        2. Select the Carrier Guest page in the Carrier Guest Email Loop.

        3. Find and choose the Carrier Guest component.

          1. Set the value for the Load Results List Size field. The default value is 25. This value controls the number of Loads displayed in the RFQ Load Board LWC per page.

          2. The following values may be set.

            1. 5

            2. 10

            3. 25

            4. 50

            5. 100

          3. Publish the page when complete.

      4. From Setup, search for and select Permission Sets.

        1. Find and select the permission set controlling the Carrier Email Loop. (The correct permission set was cloned from the TMS Carrier Email Loop.)

        2. Choose the Object Settings link.

        3. Click the link for the Accessorial Exclusions object.

        4. Choose Edit.

        5. Under Object Permissions, Check the Enabled box for Read access.

          1. Under Field Permissions, Check Read Access for each field on the object.

        6. Click Save.

        7. Return to the Object Settings link.

        8. Click the link for the Carrier Exclusions object.

        9. Choose Edit.

        10. Under Object Permissions, Check the Enabled box for Read access.

          1. Under Field Permissions, Check Read Access for each field on the object.

        11. Click Save.

        12. Return to the Object Settings link.

        13. Click the link for the Zones object.

        14. Choose Edit.

        15. Under Object Permissions, Check the Enabled box for Read access.

          1. Under Field Permissions, Check Read Access for each field on the object.

        16. Click Save.

    18. For customers using the MyCarrierPackets integration, complete the following steps.

      1. Add the Onboard Carrier LWC to the Home page. Or another page as requested. NOTE: Steps a, b, and c may be completed together.

        1. On the Home page, select Setup → Edit Page.

        2. Add the Onboard Carrier component to the page.

      2. Update the Carrier Equipment Type related list.

        1. Scroll to the Carrier Equipment Types related list and select it.

        2. In the list options, find the Related List Fields section.

        3. Remove the Carrier Equipment Type: Name field.

        4. Click the Add Field button and choose the Equipment Type: Name field.

      3. Add new fields to the Carrier Transportation Profile Lightning Record Page.

        1. Add the following fields to the Lightning Record page with the other Motor Carrier Analysis section (MCP fields).

          1. Other Risk

          2. Other Risk Points

      4. Click Save.

        The Other Risk assessment type on the Transportation Profile does not factor into the packaged TMS Auto-Reject Carrier Workflow Rule.

        Salesforce no longer supports creating Workflow Rules. Incorporate these fields as desired by creating a new Flow.

    19. Add new EDI Rail support fields to the following page layouts.

      1. Add the following fields to the appropriate page layouts.

        1. From Setup → Object Manager navigate to the Line Item object.

          1. Select the Page Layouts tab. Choose the Line Item page layout.

          2. Add the STCC Code field to the page layout.

          3. Click Save.

        2. Navigate to the Load object.

          1. Select the Page Layouts tab. Choose the Load page layout.

          2. Add the Equipment Weight, Equipment Weight Units, Seal Number, and Waybill Number fields to the page layout.

          3. Click Save.

        3. Navigate to the Mode object.

          1. Select the Page Layouts tab. Choose the Mode page layout.

          2. Add the Rail field to the page layout.

          3. Click Save.

    20. Fleet Management Setup: complete the following steps for Fleet customers as necessary.

      1. To view line item information on the Trip Planner LWC, enable the button from TMS Admin → Configuration.

        1. Under Global Settings, find Show Line Item Button on Trip Plan Modal and Check the box.

      2. Add the Distance Units in Trip Planner to the User page layout.

        1. From Setup → Object Manager, select the User object.

        2. Select the User Page Layouts tab.

        3. Add the Distance Units in Trip Planner field to the page layout.

        4. Click Save.

      3. Navigate to each Fleet user and select the preferred distance unit. Miles and Kilometers (default selection), Miles, or Kilometers.

      4. Enable IFTA mileage and fuel reporting by completing the following steps.

        1. Set the following fields on TMS Admin → Configuration → Fleet Management.

          1. Check the IFTA Mileage Reporting Enabled field.

          2. Set the IFTA Off Route Miles field to the appropriate precision value. The default value is 0.03. If no value is set in the field, The TMS uses the default value.

        2. Schedule the IFTAStateReportJob from the Batch Process Monitor.

          1. The first time running this job back-populates all Trip Plan Segment records with a Status of Completed and Pending Final. This job runs at midnight each night. Avoid hitting Salesforce governor limits for orgs with many Trip Plan Segments by selecting Schedule, not Run Now.

      5. Enable the Driver Confirmation enhancements by Checking the following fields under TMS Admin → Configuration → Fleet Management.

        1. Email Modal for Driver Confirmation

        2. Driver Decline Reason Required

      6. For Catena & Samsara ELD integration users, enable Feed Tracking for the Driver object. This enables the Chatter feature and allows for Driver messaging across the Fleet Management Module.

        • From Setup → Feed Tracking, select the Driver object.

        • Check Enable Feed Tracking.

        • Click Save.

    21. Make sure all required TMS batch jobs are running and/or scheduled. See the Batch Processes article for suggested run times.

    Thoroughly test ALL EDI Transactions in the sandbox before upgrading to production. EDI enhancements included in the Spring ‘25 release impacts all EDI transactions.
